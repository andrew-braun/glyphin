# Homepage Simplification + Language Picker Plan

## Objective

Unify homepage behavior across signed-in and signed-out learners, remove the current orbit illustration from homepage usage, and replace hero-side artwork/value-prop content with a scrollable language/alphabet picker that emphasizes Thai as the only active option.

## Current State Snapshot

- `src/routes/+page.svelte` conditionally renders two different top-level home experiences:
  - learner path: `LearnerHomeHub` + stats + lesson list
  - guest path: `GuestHomeShell` + `HowItWorksSection`
- `GuestHomeShell` and `LearnerHomeHub` both currently import and render `GlyphOrbit`.
- `GuestHomeShell` currently contains:
  - target CTA copy (already present)
  - value proposition bullet list (`valueProps`) the request wants removed
  - two CTA buttons (`Start Thai free`, `Save progress later`) that do not match requested behavior
  - a non-scrollable language card grid that includes only 3 languages and mixes statuses/content format.

## Implementation Plan

### 1) Consolidate homepage entry state into a single hero experience

- Update `src/routes/+page.svelte` to stop splitting between guest and learner shell variants for the hero.
- Keep one shared hero component for both states (recommended: adapt and reuse `GuestHomeShell` so we avoid introducing parallel abstractions).
- Pass only the minimum state needed for CTA labeling/target behavior into that shared hero:
  - `authenticated` boolean from `$authSession.authenticated`
  - optional `continueHref` (likely `/learn/${$currentLessonId}`) if authenticated flow should resume directly.
- Keep lower-page learner-specific sections (`HomeStatsOverview`, `LessonList`) behavior as-is unless intentionally changed; this keeps scope focused on “new user vs current user state” for the top/home entry experience while preserving progress utilities.

### 2) Remove `GlyphOrbit` from homepage composition without deleting component

- Remove `GlyphOrbit` imports/usages from homepage hero components (`GuestHomeShell`; and `LearnerHomeHub` if still used post-step-1).
- Keep `src/lib/components/illustrations/GlyphOrbit.svelte` unchanged and committed in place for future reuse.
- Verify no dead imports remain after layout consolidation.

### 3) Replace hero body content to match requested CTA model

- In the shared hero component:
  - Keep the exact CTA text block:
    - Heading: “Pick a language. Start in under a minute.”
    - Body: “Glyphin teaches script and vocabulary together, so your first lesson already feels like reading the real thing.”
  - Remove value proposition list markup and data (`valueProps`).
  - Replace multi-button action group with a single primary CTA button whose label and destination depend on auth state:
    - signed out: `Sign Up` → `/auth`
    - signed in: `Continue` → `/learn/${currentLessonId}` (or agreed continue route)
- Ensure button wording uses title case exactly as requested.

### 4) Build right-side scrollable language/alphabet selector in hero

- Replace the orbit-art column with a vertical, scrollable list panel rendered on the right side of the hero.
- Define a typed language config array in the hero component with all requested entries:
  - Thai (active/clickable)
  - Korean
  - Cyrillic
  - Greek
  - Arabic
  - Georgian
  - Japanese (Kana)
  - Devanagari
  - Hebrew
  - Armenian
  - Amharic
  - Vietnamese
  - Turkish
  - Icelandic
  - Burmese
- For each row render:
  - alphabet/language name
  - sample glyphs/words (short static examples per script)
  - right-aligned action:
    - active row (Thai): enabled `Learn Thai` button linking to `/learn/1`
    - inactive rows: disabled control and clear inactive affordance (e.g., muted row + “Inactive” badge/text)
- Implement container sizing so approximately 4–5 rows are visible before scrolling:
  - use a fixed/clamped max-height for the list container
  - `overflow-y: auto`
  - preserve touch scrolling behavior on mobile.
- Keep responsive behavior:
  - desktop/tablet: copy left, list right
  - smaller breakpoints: stacked layout with list below copy, preserving scrollbox semantics.

### 5) Styling + accessibility pass

- Keep semantic structure (`section`, heading hierarchy, list semantics where appropriate).
- Ensure disabled/inactive rows are visibly and programmatically non-interactive (`disabled`, `aria-disabled`, no active href behavior for inactive entries).
- Preserve existing tokenized design system usage (`$space-*`, color vars) rather than ad-hoc styles.
- Confirm keyboard navigation/focus behavior in scroll container and for the active Thai button.

### 6) Cleanup and validation

- Remove now-unused modules/imports (e.g., `HowItWorksSection` import/usage if the unified homepage no longer needs it in this scope).
- Run targeted checks:
  - `pnpm check` (required for non-trivial change)
  - optionally `pnpm lint` if check output points to route/component warnings.
- Manual verification via `pnpm dev`:
  - signed-out homepage shows unified hero with `Sign Up` button
  - signed-in homepage shows same hero with `Continue` button
  - no orbit artwork in homepage
  - scroll list displays all requested languages, Thai only active
  - approximately 4–5 rows visible before scrolling on standard desktop viewport.

## File-Level Change Map (expected)

- `src/routes/+page.svelte`
  - unify top-level branch behavior and pass auth/current lesson props to shared hero
- `src/lib/components/content/home/GuestHomeShell.svelte`
  - become shared home hero; remove value props and orbit; add scrollable language list; add auth-driven CTA
- `src/lib/components/content/home/LearnerHomeHub.svelte`
  - potentially removed from route usage (component file may remain untouched unless explicitly pruning)
- `src/lib/components/content/home/HowItWorksSection.svelte`
  - potentially removed from homepage composition if no longer part of the new simplified flow
- `src/lib/components/illustrations/GlyphOrbit.svelte`
  - no changes (kept for future reuse)

## Risks / Decisions to lock before implementation

- Route target for `Continue` when signed in:
  - recommended: `/learn/${currentLessonId}` to preserve existing continuity behavior.
- Whether to retain learner-only lower sections (`HomeStatsOverview`, `LessonList`) after unified hero:
  - recommended: keep as-is for now; this satisfies homepage-state unification at the hero while minimizing scope.

## Done Criteria

- Homepage hero is structurally the same for signed-in and signed-out states.
- No homepage usage of `GlyphOrbit`; component still exists in repo.
- Hero contains only requested CTA copy + one auth-conditional CTA button.
- Right side hero panel is a scrollable language list showing requested scripts, Thai as sole active option.
- Build/check passes and UI behavior verified for desktop/mobile layouts.
