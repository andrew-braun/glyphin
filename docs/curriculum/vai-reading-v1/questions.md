# Vai Reading Questions

Use this file for unresolved decisions that research cannot safely settle during
the `vai-reading-v1` bootstrap.

## Architecture

- How should the runtime represent a large syllabary inventory with series and
  nasalized-sign metadata?
- Should placeholder sequence rows be allowed in candidate CSVs or should a
  separate reviewer-needed state exist?
- Does the course need a source-confidence field surfaced in authoring UI before
  implementation?

## Product And Pedagogy

- Which Vai dialect and spelling conventions should define v1?
- Should lessons start from a script-name anchor or from reviewer-selected common
  words?
- Which target domains are appropriate for a first public course?

## Sources, Licensing, And Attribution

- Confirm terms for scholarship and digital text before using examples beyond
  discovery.
- Keep all external examples out of shipped content until license and reviewer
  approval are complete.
- Prefer app-authored examples reviewed by Vai language and script experts.

## Reviewers And Validation

- Assign Vai reviewers for spelling, pronunciation, domain fit, and cultural
  context.
- Test Noto Sans Vai or approved alternatives at lesson and drill sizes.
- Replace placeholder anchors with reviewed words before runtime publication.

## App Expansion Recommendations

- Add syllabary metadata for large inventories and series grouping.
- Add explicit reviewer-needed status for low-resource candidate rows.
- Add font QA checks for visually dense syllabary signs.
