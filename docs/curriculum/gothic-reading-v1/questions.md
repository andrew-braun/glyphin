# Gothic Reading Questions

Use this file for unresolved decisions that research cannot safely settle during
the `gothic-reading-v1` bootstrap.

## Architecture

- How should Gothic script text, transliteration, pronunciation notes, and
  manuscript edition notes be stored separately?
- Should sacred-context metadata be shared with Glagolitic and Coptic courses?
- How should SMP font fallback checks be integrated into curriculum review?

## Product And Pedagogy

- Should v1 choose biblical manuscript recognition as the explicit domain or
  present Gothic as a broader historic-language course?
- Which sacred terms can be used early without overloading cultural context?
- Should pronunciation be visible before a specialist-approved reconstruction
  policy exists?

## Sources, Licensing, And Attribution

- Confirm Wulfila Project and TITUS reuse terms before deriving examples.
- Keep Wiktionary discovery-only until CC BY-SA attribution and share-alike
  implications are reviewed.
- Prefer app-authored short examples for shipped content.

## Reviewers And Validation

- Assign a Gothic or historical Germanic specialist to validate anchors,
  transliteration, pronunciation hints, and cultural framing.
- Spot-check early anchors against manuscript and lexicon contexts.
- Verify Gothic font rendering at drill sizes on desktop and mobile.

## App Expansion Recommendations

- Add historic-language and sacred-context metadata.
- Add transliteration and reconstructed-pronunciation fields.
- Add edition/source notes for manuscript-derived examples.
- Add SMP font fallback checks for Gothic.
