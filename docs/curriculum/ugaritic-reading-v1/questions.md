# Ugaritic Reading Questions

Use this file for unresolved decisions that research cannot safely settle during
the `ugaritic-reading-v1` bootstrap.

## Architecture

- How should the runtime model store consonant-only Ugaritic text with
  transliteration, gloss, and optional reconstructed vocalization?
- Should word dividers be modeled like punctuation, grapheme units, or explicit
  segmentation rules?
- How should aleph variants and ayin be represented for learner drills?

## Product And Pedagogy

- Should v1 teach the full alphabet chart or sequence by short tablet words?
- How should divine and mythological anchors be framed so they are not reduced to
  generic vocabulary?
- Should pronunciation hints be avoided until a specialist-approved policy is in
  place?

## Sources, Licensing, And Attribution

- Confirm corpus access and reuse terms before deriving examples from Ugaritic
  databases.
- Keep Wiktionary and reference sites discovery-only until attribution and
  derivative-use obligations are reviewed.
- Prefer app-authored short sequences for all shipped examples.

## Reviewers And Validation

- Assign a Northwest Semitic or Ugaritic specialist to validate sign names,
  transliteration, glosses, and sacred context.
- Spot-check anchors against real tablet domains before final sequencing.
- Verify Ugaritic font rendering and word-divider visibility at drill sizes.

## App Expansion Recommendations

- Add consonantal-script transliteration fields and optional vocalization notes.
- Add word-divider drill support for historic scripts.
- Add sacred or culturally sensitive anchor metadata.
- Add SMP font fallback checks for Ugaritic.
