# Cherokee Reading Questions

Use this file for unresolved decisions that research cannot safely settle during
the `cherokee-reading-v1` bootstrap.

## Architecture

- How should the runtime represent syllabary-scale inventories without forcing
  alphabet-style letter categories?
- Should Cherokee lowercase be stored as case variants on one grapheme row or as
  separate display forms in course-version metadata?
- How should app search and drill matching handle uppercase-only source material
  versus modern mixed-case Cherokee text?

## Product And Pedagogy

- Which dialect and orthographic conventions should v1 follow?
- Should the course start with greeting and identity words or with a neutral
  chart-family path selected by Cherokee educators?
- How much cultural framing is required before learner-visible examples can ship?

## Sources, Licensing, And Attribution

- Confirm terms for Cherokee Nation language materials before using them beyond
  discovery.
- Keep Wiktionary-derived forms discovery-only until CC BY-SA obligations are
  reviewed.
- Prefer app-authored examples reviewed by Cherokee language experts for shipped
  content.

## Reviewers And Validation

- Assign Cherokee language reviewers for spelling, pronunciation, dialect, and
  cultural context.
- Verify target fonts render Cherokee and Cherokee Supplement clearly at drill
  sizes.
- Confirm whether example domains should include public signage, classroom text,
  dictionary headwords, or all three.

## App Expansion Recommendations

- Add syllabary-friendly grapheme metadata and course-specific chart groupings.
- Add case-variant support that is not tied to Latin casing assumptions.
- Add a reviewer-status field for culturally sensitive or community-authored
  anchors.
