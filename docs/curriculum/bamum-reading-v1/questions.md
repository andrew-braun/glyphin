# Bamum Reading Questions

Use this file for unresolved decisions that research cannot safely settle during
the `bamum-reading-v1` bootstrap.

## Architecture

- How should script-stage metadata distinguish modern Bamum block signs from
  Bamum Supplement historical-stage signs?
- Should placeholder sequence rows be allowed in candidate CSVs or should a
  separate reviewer-needed state exist?
- Does the course need museum or historical-context fields before publication?

## Product And Pedagogy

- Which Bamum script stage should define v1?
- Should lessons start from a script-name anchor or from a sign-family path while
  real anchors are reviewed?
- Which target domains are appropriate for a public course without overclaiming
  modern daily use?

## Sources, Licensing, And Attribution

- Confirm terms for scholarship and museum references before using examples
  beyond discovery.
- Keep all external examples out of shipped content until license and reviewer
  approval are complete.
- Prefer app-authored examples reviewed by Bamum script experts.

## Reviewers And Validation

- Assign Bamum reviewers for script stage, spelling, pronunciation, and cultural
  context.
- Test Noto Sans Bamum or approved alternatives at lesson and drill sizes.
- Replace placeholder anchors with reviewed words before runtime publication.

## App Expansion Recommendations

- Add script-stage metadata for historical scripts with multiple encoded stages.
- Add explicit reviewer-needed status for low-resource candidate rows.
- Add museum or heritage context fields for historically sensitive lessons.
